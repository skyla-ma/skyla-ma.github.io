Scaling Monosemanticity — Technical Summary

Original post: https://transformer-circuits.pub/2024/scaling-monosemanticity/

The goal of the paper is to address polysemanticity, where individual neurons respond to many unrelated patterns, making internal model representations hard to interpret. The paper talks about separating these mixed signals into features that correspond more cleanly to specific concepts.

Method

They apply sparse autoencoders (SAEs) to activations from a layer of Claude 3. The autoencoder is trained to reconstruct the original activations, but under a sparsity constraint: only a small number of learned features can be active at a time. Thus, the model represents each activation vector as a combination of a few elements from a large learned dictionary.

Because only a limited number of features are allowed to fire, the system is encouraged to represent signals using more specific and distinct components rather than blended ones. Each learned feature corresponds to a direction in activation space that contributes to reconstructing the original signal.

Findings

Scale

The method is applied to Claude 3, producing millions of learned features. They had already demonstrated similar behavior in smaller transformer models, but in this paper they extend that to a large production model.

Granularity

Many learned features correspond to specific patterns in data, such as:

  • programming concepts and error types
  • named entities and places
  • transportation systems
  • medical and scientific topics
  • social and behavioral patterns such as persuasion or deception

These features activate selectively when related content appears in the input.

Causal testing

To test whether features influence model behavior, they intervene during inference by forcing certain features to remain active (clamping). This changes the model’s outputs in ways consistent with the feature’s associated concept. The interventions show that these features participate in generating behavior, not just correlating with it.

Coverage and limitations

  • Not all learned features are interpretable.
  • Some features rarely activate (dead features).
  • Some still combine multiple concepts rather than representing a single one.
  • Training SAEs and running them across layers is computationally expensive.

Overall summary

Sparse autoencoders can decompose internal activations of a frontier language model into a large set of sparse features that correspond to identifiable patterns in data. These features can be measured, analyzed, and causally manipulated, providing a method for studying internal representations at scale.
